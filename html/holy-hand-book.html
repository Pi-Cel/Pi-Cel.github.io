Tip 1 - Change IDE to JetBrains Rider.

This will set JetBrains Rider as your IDE for Unity instead of Visual Studio.

Requirements - JetBrains Rider

1 - Open up Unity

2 - Go to Edit > Preferences > External Tools

3 - Click on the External Script Editor and choose to Browse...

4 - To find the location of Rider, enter Rider into the Search Bar

5 - Keep on Right Click > Open file location until you get to rider64.exe

Unity should now open your C# files using JetBrains Rider.

Tip 2 - Change the playmode tint.

This will help you know when you're in playmode, and trust me, it's a life / work saver.

Requirements - None

1 - Open Unity.

2 - Go to Edit > Preferences > Colours.

3 - Change the Playmode under General > Playmode tint.

4 - Edit the tint in playmode to see it change live.

Now when you enter playmode, Unity's UI will be tinted.

Tip 3 - Change UI to Dark Theme.

This will give Unity's UI a dark tint, which is a paid feature.

Requirements - HxD

1 - You'll need to locate Unity.exe

1.1 - The fastest way to do this is by finding the Unity shortcut on the desktop

1.2 - Right click the Unity Shortcut > Open File Location > Unity.exe

1.3 - If you don't have a Unity shortcut on the desktop, enter Unity.exe into the Search Bar

2 - Make a backup of Unity.exe just in case you mess up

3 - Copy it to the desktop and open it using HxD

4 - Open up the search box by either going to Search > Find or pressing Ctrl + F

5 - Change the search type to Hex Values

5 - Paste 84 C0 75 08 33 C0 48 83 C4 20 5B C3 8B 03 48 83 C4 20 5B C3 into the Search For box

6 - Replace the third value Should be 75 by default to 74 and save the file

7 - Copy the edited Unity.exe and replace the one in the Unity folder

Now when you load up Unity, a dark theme will be displayed.

~ Variables ~

Variables are like containers, they can store different types of data. 

              // int is a full number.
              int playerHealth = 123;

              // float is a number with a decimal or floating point.
              float weaponRange = 123.456f;

              // double is a float with a higher level of accuracy.
              double horizontalRotation = 123.456;

              // string is a list of letters, numbers, and other characters.
              string playerName = "Justin";

              // char is a single character that strings are built out of.
              char playerInitial = 'J';

              // var is a special type that will change depending on what's put on the right.
              var aNumber = 123;
              var aString = "Hello World!"

More about Variables.

~ Arrays ~

Arrays are a type of variable that can hold a set amount of values. 

              // an array is defined using type[] which means you can turn pretty much anything into an array.
              // There are multiple ways to initialize arrays, one is to declare its size.
              int[] Names = new int[10];

              // And then you can edit each one specifically.
              Names[0] = "Justinas";

              // You can also initialise them using a list.
              int[] myIntegers = new { 0, 20, 30, 123};

~ Classes ~

Every script you make in Unity will start as a class. 

              // "using" is a keyword that includes namespaces.
              using UnityEngine;

              // This is definition for a class, it also inherits from MonoBehaviour.
              public class TutorialClass : MonoBehaviour
              {
              	// Variables defined at the top are called Fields.
              	private int _myNumber = 10;

              	// Start is one of the functions that comes from MonoBehaviour.
              	private void Start()
              	{
              		print(_myNumber);
              	}
              }

More about Classes.

~ Fields ~

More about the variables that are defined at the start of the class. 

            // First of all, fields can't be of type var, and therefore have to be set.
            int maxHealth = 10;
            // Keep your fields private if you can, and create getter and setter functions instead.
            private int _currentHealth = 2;

            // Private variables won't show up in the editor, so you can use the Serialize Field attribute.
            [SerializeField] private int _damage = 10;

            // Getter and Setter functions.
            public void GetCurrentHealth()
            {
            	return _currentHealth;
            }

            public int SetCurrentHealth(int currentHealth)
            {
            	_currentHealth = currentHealth;
            }

~ Attributes ~

Attributes are like the cherry on top, they modify variables in various ways. 

            // Headers create a space and a bold h1 in the editor.
            [Header("Variables!")]

            // SerializeField will force a field to show up in the editor, even if it's private.
            [SerializeField] private bool _isRightHanded;

            // Space will create a gap between the variables in the editor.
            [Space]

            // Range will turn a numerical field into a slider.
            [Range(-10, 10)] public float sliderField = .1f;

            // Tooltip will make a description appear when you hover over a field.
            [Tooltip("The force that is applied to the character when he jumps.")] public float JumpForce = 300;

            // Attributes can also be combined to save space.
            [Space, SerializeField, Range(0, 200), Tooltip("The base accuracy of the weapon")]
            private int _weaponAccuracy;

            // RequireComponent will force the GameObject with this script to also have the required component.
            [RequireComponent(typeOf("Camera"), typeOf("CameraController"))]
            public class CameraRotation : MonoBehaviour

~ Random numbers ~

Random numbers are very usefull to, well, add randomisation to your game.

              // this will generate a random integer.
              var randomIntNumber = Random.Range(0, 100);

              // As long as at least one parameter is a float, the result will also be a float.
              var randomFloatNumber = Random.Range(-10f, 100);

~ Getters and Setters ~

Private objects can't be seen by other classes (hence the name) the way we get around this is by using getters and setters.

              // We don't want other scripts to directly change the health of the player.
              [SerializeField] private int _health;

              // You instead create functions that modify the variables, which also allows you to do other things on top.
              // This is a setter which modifies (sets) a variables.
              public void Heal(int amount)
              {
              	_health += amount;
              	// Do other stuff too.
              }

              // Getters return a variable, so that other classes can see it.
              public int GetHealth()
              {
              	return _health;
              }

~ Programs you should have ~

Unity - 3D / 2D Games Engine - Unity is the most popular, free to use Game Engine around, and it deserves that title.

GameMaker: Studio - 2D Games Engine - Game Maker is a game engine used to soley make 2D games, and as such, it probably is the best Game Engine for 2D games.

Godot - 3D / 2D Games Engine - Godot is an open source engine similar to Unity.

LÖVE - 2D Games Engine - Love is a lightweight, free to use, 2D Games Engine that's quite good for smaller, simpler games, I used this to learn Lua.

Pygame - 2D Games Engine - Python equivalent of LOVE. I've make snake, really fun!

Construct 3 - 2D Games Engine - Construct 3 is a Games Engine that runs in your browser, looks pretty cool.

Amazon Lumberyard - 3D Games Engine - I haven't really heard much about this Games Engine, since it seems to be in beta, but considering that it's made by Amazon, and is used to make Star Citizen, I feel like this is going to be a big one.

Aseprite - Pixel Art Editor - Aseprite is probably the best pixel art tool, Photoshop is for noobs with money. You can either buy it, or compile it for free from Github, or ask me in person.

GrafX2 - Pixel Art Editor - This is a super retro pixel art editor that used to be very popular back in the day, but it's mostly become obselete, though still is a great editor.

Graphics Gale - Pixel Art Editor - A free alternative to Aseprite, and trust me, this is an awesome free alternative, people say it's better for animation, but it's quite a retro tool.

PixelShop - Pixel Art Editor - Quite similar to Graphics Gale in it's UI, but it has many features that make it stand out.

Pyxel Edit - Pixel Art Editor - Yet another pixel editor, works in tiles which makes it really useful when creating tilesets.

Notepad++ - Text Editor - A very lightweight IDE, just like the name suggests, it's a notepad on drugs, and should be used instead of notepad.

Atom - Text Editor - A relatively heavyweight IDE that can be used for pretty much everything.

Visual Studio Code - Source Code Editor - A lightwight, open source source code editor that runs everywhere.

Sublime Text - Text Editor - Sublime is an IDE that's the best of the both worlds, quite lightweight, but also makes life much easier.

Brackets - Source Code Editor - If you, like me, like to build webpages from scratch, this might be the wonder tool you're looking for.

JetBrains Rider - IDE - An IDE targeted for C#, It has probably the best Unity plugin which makes writing scripts much easier, trust me.

GitHub Desktop - GUI for Git - Makes managing and uploading your projects super easy, GitHub by default can be quite hard to use, this makes it much easier.

Crocotile 3D - Modelling Software - Use this to make 3D models that use 2D tiles, it produces awesome looking models.

MagicaVoxel - Voxel Art Editor - This is a free voxel art editor, my personal favourite!

VoxelShop - Voxel Art Editor - This is an open source voxel art modeller and runs on pretty much everything, Has some features that MagicaVoxel doesn't.

Qubicle - Voxel Art Editor - This is the voxel art tool that Crossy Road was made in, really good, but it ain't cheap.

Bocsca Ceoil - DAW - Use this to make 8-bit / chiptune music for your game. Here's Tutorial by Brackeys.
